# Coverage Manifest, File Classification, micro_per_hunk, Judge & Verifier Contract

> This module is read by **Phase 5** (`micro_per_hunk`), **Phase 7.5** (Judge / Verifier),
> and **Phase 8** (Coverage Manifest + external verifier) of `/review`.
>
> **The manifest is the proof-of-work.** It is written to
> **`.claude/review-manifest-{ts}.json`** (`{ts}` = `$(date +%Y%m%dT%H%M%SZ)` UTC). A
> non-LLM script, `~/.claude/scripts/review-verify-manifest.sh`, recomputes hunks,
> symbols, and `diff_hash` from `git` and **INVALIDATES the run on any mismatch**. The model
> may NOT self-attest coverage: an APPROVE without a verifier-passed manifest is an invalid
> run. This is what makes fake-pass mechanically detectable.

---

## 1. File Classification (Phase 0.6 — defines the coverage denominator)

Every file in `diff.files` MUST be tagged with exactly one `file_class` BEFORE macro/micro.
The class decides what coverage the verifier demands for that file. First matching rule wins,
top to bottom.

```yaml
file_class:
  enum: [code, config, iac, generated, vendored, binary, rename, lockfile, docs]

  detection:                       # evaluated in order; first match wins
    binary:
      rule: "git diff --numstat shows '-' '-' (non-text blob)"
      probe: "git diff --numstat \"$BASE...$HEAD\" -- <f> | awk '$1==\"-\"&&$2==\"-\"'"
    rename:
      rule: "git diff status R### with 100% similarity (pure rename, no content change)"
      probe: "git diff --find-renames --name-status \"$BASE...$HEAD\" | rg '^R100\\b'"
    lockfile:
      rule: "package-lock.json|yarn.lock|pnpm-lock.yaml|Cargo.lock|go.sum|poetry.lock|Gemfile.lock|composer.lock"
    generated:
      rule: "path matches *_generated.*|*.pb.go|*.pb.*|*_pb2.py|openapi*/|*.gen.* OR first 5 lines contain 'DO NOT EDIT'/'autogenerated'/'@generated'/'Code generated by'"
      probe: "head -5 <f> | rg -qi 'do not edit|autogenerated|@generated|code generated by'"
    vendored:
      rule: "path under vendor/|node_modules/|third_party/|.venv/|Pods/|.cargo/"
    iac:
      rule: "*.tf|*.tfvars|*.hcl OR path under k8s/|helm/|charts/|.github/workflows/|ansible/ OR *.yaml/*.yml that is a k8s/helm/compose/CI manifest OR Dockerfile|*.dockerfile"
    config:
      rule: "*.json|*.yaml|*.yml|*.toml|*.ini|*.env|*.properties|*.conf NOT already iac/lockfile/generated"
    docs:
      rule: "*.md|*.rst|*.txt|*.adoc|docs/** NOT generated"
    code:
      rule: "default — any source extension routed to a specialist (Phase 6) not matched above"

  coverage_rule:                   # what the verifier enforces per class
    code:                  "macro_pass=true AND micro_pass via micro_per_function REQUIRED."
    config|iac:            "macro_pass=true AND micro_pass via micro_per_hunk REQUIRED."
    # sql/schema/protobuf/Dockerfile/yaml are carried as `kind` inside an iac/config file's hunks
    generated|vendored|binary|lockfile|rename:
      "macro_pass=light; micro_pass = \"N/A (<class> — <one-line justification>)\".
       lockfile ADDITIONALLY requires a supply-chain tier hit (osv-scanner/trivy/govulncheck)."
    docs:                 "macro_pass=true; micro_pass = docs-accuracy check or \"N/A (docs)\"."
```

**Class affects the verifier directly:** Check 4 (below) fails the run if any `code`,
`config`, or `iac` file has `macro_pass=false` or a missing/false `micro_pass`. Marking a
real code file as `generated` to dodge micro is detectable — the verifier re-derives the
class from the same rules and from `git diff --numstat`/`--name-status`.

## 2. micro_per_hunk Schema (Phase 5 — non-function files)

`code` files get `micro_per_function` (see the spec's Phase 5 block). Every `config` and
`iac` file gets `micro_per_hunk` instead — one entry per changed hunk, each carrying a
`kind` so the right checklist and owner apply. No non-function hunk ships un-inspected.

```yaml
micro_per_hunk:                    # for config | iac | sql | schema | protobuf | dockerfile | yaml
  - hunk: <file:line-range>        # MUST map to a real @@ hunk in git diff
    kind: "iac | k8s | sql | protobuf/asn1 | dockerfile | ci | config | yaml"
    owner: developer-executor-security + <relevant specialist>   # devops-specialist-* for IaC
    checks: [...]                  # the per-kind checklist below, applied and recorded
    finding_ids: [...]             # findings emitted; OR an explicit clean: line (see §3)

  per_kind_checklist:
    iac:        "secret material in plaintext; privilege/scope creep (IAM *:*, 0.0.0.0/0);
                 public exposure; missing encryption-at-rest; default drift; state/backend safety;
                 resource limits; provider version pinning."
    k8s:        "privileged/hostPath/hostNetwork; missing resource limits/requests; :latest tags;
                 missing securityContext/runAsNonRoot; secrets as env vs mounted; RBAC over-grant;
                 missing liveness/readiness."
    sql:        "destructive/irreversible migration; lock-heavy op on large table; additive-vs-breaking;
                 missing down migration; injection in dynamic SQL; index implied by new predicate;
                 NOT NULL without default on populated table."
    protobuf/asn1: "field-number reuse/renumber (wire break); type change on existing field;
                 required<->optional drift; enum value reuse; package/option change;
                 ETSI/3GPP wire-format conformance where relevant."
    dockerfile: "unpinned base image (:latest / no digest); running as root; secret in ARG/ENV;
                 ADD remote URL; missing USER; cache-busting layer order; no HEALTHCHECK where expected."
    ci:         "untrusted input in run: (script injection); secrets echoed; overbroad permissions:;
                 unpinned action (use SHA, not @v); pull_request_target risk; missing concurrency guard."
    config|yaml:"schema validity; secret material; default drift changing behavior; breaking key
                 rename/removal consumers rely on; environment leakage; type coercion footguns."
```

The `kind` selects the deterministic tier too (Phase 3): `iac`→checkov/tflint/tfsec,
`k8s`→kube-linter/kubeconform, `sql`→sqlfluff, `protobuf`→buf, `dockerfile`→hadolint,
`ci`→actionlint, `yaml`→yamllint. Tier hits dedupe into the matching hunk's `finding_ids`.

## 3. Judge / Verifier Protocol (Phase 7.5 — a separate stage, guards BOTH directions)

A dedicated Judge agent re-reads **every** surviving finding **against the cited code** and
re-grounds it with a recorded command trail. The Judge guards against false positives
(hallucinated bugs) AND false negatives (theatrical "all clean").

```yaml
judge_protocol:
  per_finding:
    1_reground:
      action: "Re-open the cited file:line with Read; re-run the grounding command
               (rg / ast-grep / linter) and CAPTURE its output into the finding's trail."
      record: "finding.evidence_trail = {command, output_excerpt, cited_file_line}"
    2_classify:
      grounded:
        when: "cited line exists AND says what the finding claims AND counterexample holds"
        action: "keep as Confirmed."
      ungroundable:
        when: "plausible but the Judge cannot confirm with a concrete command/line
               (isolation infeasible, repro not run, weak signal)"
        action: |
          DEMOTE to Needs-Verification — NEVER silently delete.
          Attach an explicit note: "could not confirm because <reason>".
          (A CRITICAL/HIGH that lands here GATES: caps score <=3, blocks --loop exit.)
      hallucination:
        when: "cited line does NOT exist, OR the code says the OPPOSITE of the claim"
        action: "DROP — the ONLY case where a finding is removed.
                 Log it in the suppression appendix with the reason + the cited (wrong) line."

  hard_rules:
    - "Demotion is the default for weak evidence; deletion is reserved for hallucinations only."
    - "'A grep returning nothing is NOT proof of a bug.' An empty grep can neither create
       nor confirm a finding — it is recorded as inconclusive, not as evidence."
    - "Every drop AND every demotion is logged (what + why + cited line). Silent discard is forbidden."

  false_negative_crosscheck:       # the anti-theater half of the Judge
    canary:
      action: "Read Phase 0.8 canary result from the manifest."
      rule: "canary == failed  =>  an EMPTY findings array becomes INCONCLUSIVE
             (the engine is mis-calibrated; 'no findings' is untrustworthy)."
    per_dimension_clean:
      action: "For every dimension marked checked:true in micro_per_function / micro_per_hunk,
               require EITHER a finding_id OR an explicit clean: justification citing a real line
               (e.g. clean: \"inputs validated at foo.go:42, no taint reaches sink\")."
      rule: |
        checked:true with NO finding and NO cited clean: line
          =>  that dimension reverts to `unverified`
          =>  the verdict CANNOT be APPROVE.
      kills: "unfalsifiable check-marks — a checked box with no owner output or cited line is theater."

  ensemble:                        # high-stakes / --deep diffs (N=3 shuffled passes)
    rule: "finding present in >1 pass -> full confidence; single-pass -> Needs-Verification."
```

The Judge writes its outcomes back into the manifest: demoted findings carry
`verification: "Needs Verification"` + note; dropped hallucinations are listed under
`suppressed[]` with `reason`. Confidence is recomputed from objective signals
(tool-confirmation, ensemble agreement, working POC) — never free-lowballed.

## 4. Coverage Manifest JSON Schema (Phase 8)

Written to **`.claude/review-manifest-{ts}.json`**. Compact form: counts + ranges, never
full source echoes. Every field below is required unless marked optional.

```yaml
coverage_manifest:                 # .claude/review-manifest-{ts}.json
  diff_hash: <sha256>              # = git diff "$BASE...$HEAD" | sha256sum   (verifier recomputes; mismatch => INVALID)
  base: <sha>                      # $BASE commit
  head: <sha>                      # $HEAD commit
  canary: passed | failed          # Phase 0.8 self-test result
  hunks_total: <n>                 # total changed hunks across all files (verifier recomputes; mismatch => INVALID)

  files:
    - path: <file>
      file_class: <enum>           # §1 — verifier re-derives and compares
      hunks: <n>                   # changed hunks in THIS file (sum over files == hunks_total)
      macro_pass: true             # Phase 4 macro record exists for this file
      micro_pass: true | "N/A (<class> — <justification>)" | "deferred(<reason>)"
      dimensions:                  # per-dimension status for this file
        correctness: checked
        security:    checked
        concurrency: checked
        error_handling: checked
        perf:        checked
        architecture: checked
        maintainability: checked
        idioms:      checked
        api_contract: checked
        testing:     checked
        docs:        checked
        observability: checked
        data_privacy: checked
        licensing:   checked
        portability: N/A           # only "checked" when Phase 0.7 cross_platform == true
        a11y_i18n:   N/A           # only "checked" when diff touches UI / user-facing strings
      symbols_inspected:           # MUST be a superset of the verifier's diff-extracted symbol set
        - "<pkg.Func@file:line>"
        - "<Class.method@file:line>"

  tiers:                           # generated from $DET/_table.tsv — NEVER hand-authored
    - {tool: <name>, status: "ran|absent|na|failed", exit: <int|null>, findings: <int|null>}

  blast_radius_done: true          # Phase 3.5 call/dependency graph completed
  change_coupling_done: true       # Phase 3.7 git-log co-occurrence completed
  uninspected: []                  # files/hunks not inspected; MUST be [] for an APPROVE-eligible run

  # optional, populated by the Judge (Phase 7.5)
  suppressed:                      # dropped hallucinations + logged demotions
    - {finding_id: <id>, action: "dropped|demoted", reason: <str>, cited: <file:line>}
```

**Field invariants the manifest itself must hold (the verifier re-checks all of them):**

| Invariant | Rule |
|-----------|------|
| `diff_hash` | equals `git diff "$BASE...$HEAD" \| sha256sum` |
| `hunks_total` | equals `sum(files[].hunks)` and the verifier's git count |
| `symbols_inspected` | ⊇ verifier's diff-extracted symbol set (under-enumeration FAILS) |
| `tiers[]` | each entry maps to a real `$DET/<tool>.out`; exit/findings match parsed values |
| code/config/iac files | `macro_pass=true` AND `micro_pass` true-or-valid-N/A (never false/missing) |
| `uninspected` | `[]` for any APPROVE-eligible run |
| `canary` | `passed` for any APPROVE-eligible run |

## 5. External Verifier Contract — `review-verify-manifest.sh` (Phase 8)

The run invokes the non-LLM verifier; its exit code GATES APPROVE. The model may not
define the denominator of its own coverage metric.

```bash
TS="$(date -u +%Y%m%dT%H%M%SZ)"
MANIFEST=".claude/review-manifest-${TS}.json"
# ... write $MANIFEST ...
bash ~/.claude/scripts/review-verify-manifest.sh \
  --repo "$PROJECT_DIR" --base "$BASE" --head "$HEAD" \
  --manifest "$MANIFEST" --det "$DET"
echo "verifier exit=$?"           # NONZERO => run is INVALID, regardless of model output
```

The verifier is a project script using only `git`, `jq`, and `python3` (all present in the
container — no ast-grep/tree-sitter dependency). It performs **exactly these 5 checks**:

```yaml
verifier_checks:                   # review-verify-manifest.sh — mirror EXACTLY
  1_diff_integrity:
    do:  "Recompute diff_hash = git diff \"$BASE...$HEAD\" | sha256sum, and hunks_total
          by counting @@ hunk headers across the diff."
    fail_if: "manifest.diff_hash != recomputed OR manifest.hunks_total != recomputed."
    catches: "no-op / theater runs (diff_hash mismatch) and miscounted coverage denominator."

  2_symbol_superset:
    do:  "Extract changed symbols from diff hunk headers (@@ ... @@ enclosing-decl) plus
          added/removed signature lines."
    fail_if: "manifest symbols_inspected (union over files) does NOT ⊇ the extracted set."
    catches: "under-enumeration — claiming inspection while skipping changed functions."

  3_tier_grounding:
    do:  "For each tiers[] entry, locate the captured $DET/<tool>.out and re-parse exit+count."
    fail_if: "an entry references a missing .out OR its exit/findings disagree with the .out."
    catches: "fabricated tier numbers — model-authored exit codes / finding counts."

  4_class_coverage:
    do:  "Re-derive file_class (§1 rules + git --numstat/--name-status) and check passes."
    fail_if: "any code|config|iac file has macro_pass=false OR micro_pass false/missing."
    catches: "skipped macro/micro on a real source/config/IaC file (incl. mislabeled class)."

  5_approve_eligibility:
    do:  "Gate the APPROVE-eligible state."
    fail_if: "uninspected != []  OR  canary != passed."
    catches: "green manifest with unread hunks or a mis-calibrated engine."

  exit_contract:
    "0  => manifest is internally + git-consistent; APPROVE is permitted (other gates still apply)."
    "!=0 => run is INVALID; verdict cannot be APPROVE — emit INCONCLUSIVE and surface the failed check."
```

```yaml
validity_rules:                    # verbatim from the binding spec Phase 8
  - "Verifier nonzero exit -> run is INVALID (not APPROVE)."
  - "Any code/config/iac file with macro_pass=false OR micro_pass false/missing -> INVALID."
  - "Empty findings array is valid ONLY with verifier-pass + canary-pass + per-dimension positive statement per file."
  - "An applicable deterministic tier with status=absent lowers confidence and is named, but does not alone force INVALID; status=failed with findings is hard-blocking."
```

## 6. Anti-Theater Summary (why this module exists)

| Mechanism | Defeated theater |
|-----------|------------------|
| Manifest written to `.claude/review-manifest-{ts}.json` | "I reviewed it" with no record |
| External non-LLM verifier gates APPROVE | model self-attesting its own coverage |
| Check 1 (diff_hash) | no-op / re-emitted-summary runs |
| Check 2 (symbol superset) | claiming inspection of un-opened functions |
| Check 3 (tier `.out` grounding) | hand-typed exit codes / finding counts |
| Check 4 (class coverage) | mislabeling code as generated to skip micro |
| Check 5 (uninspected/canary) | green verdict over unread hunks or mis-calibrated engine |
| Judge: demote-not-delete | dropping the hardest true positives (races, logic bugs) |
| Judge: per-dimension `clean:` line | unfalsifiable `checked:true` boxes |
| Judge: "empty grep != proof" | absence-of-evidence masquerading as evidence |

The manifest is not paperwork — it is the single artifact the external verifier reads to
decide whether the run is allowed to say APPROVE.
