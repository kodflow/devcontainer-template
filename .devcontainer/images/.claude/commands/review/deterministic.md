# Deterministic Tiers — Tool Runner (Phase 3 detail)

This is the executable detail behind **`/review` Phase 3**. Read it when you reach
"Deterministic Tiers". The contract from `review.md` is absolute and repeated here so a
budget-pressured agent cannot bypass it:

> Tools execute for real. Each writes stdout+stderr to a per-tool `.out`. The tier table
> in the report is **generated by parsing `_table.tsv` with awk**, never authored by the
> model. A tool's status is exactly one of `ran | absent | na | failed`. **`absent` is not
> `na` and neither is a pass.** Model-authored exit codes or finding counts = INVALID run
> (the external verifier, `review-verify-manifest.sh`, re-maps every `tiers[]` entry to a
> real `.out` in `$DET` and fails on mismatch).

## 1. DET workspace + the `run()` helper

`$DET` is a scratch dir; nothing here ever touches the repo. `$PROJECT_DIR`, `$BASE`,
`$HEAD` come from Phase 0 (`review-context.sh`).

```bash
DET=$(mktemp -d "${TMPDIR:-/tmp}/review-det.XXXXXX")
git -C "$PROJECT_DIR" diff --name-only "$BASE...$HEAD" > "$DET/_files.txt"

# Per-tool wall-clock cap. A hung/slow tool must NEVER stall the whole review;
# timeout sends SIGTERM at the deadline (exit 124), which run() records as a real
# nonzero exit -> derived status=failed, note 'timeout'. Override per-project /
# per-suite via REVIEW_TOOL_TIMEOUT (seconds). 180s is sane for a diff-scoped lint;
# long build/test suites get the wider REVIEW_TEST_TIMEOUT below.
RUN_TIMEOUT="${REVIEW_TOOL_TIMEOUT:-180}"
TEST_TIMEOUT="${REVIEW_TEST_TIMEOUT:-600}"

# _table.tsv columns:  invocation <TAB> status(ran|absent|na) <TAB> exit <TAB> out_lines
#   col1 is the FULL invocation ("go vet ./...") so multi-subcommand binaries (go, cargo)
#   each get one UNIQUE row + one UNIQUE .out; the verifier re-derives the slug identically.
# 'failed' is DERIVED at render time (ran + exit!=0); it is never written as a raw status.

_run_with() {                              # _run_with <timeout_s> <tool> [args...]
  local to="$1"; shift
  local tool="$1"
  local label="$*"                          # full invocation -> distinguishes go vet/build/test
  local slug="${label//[^a-zA-Z0-9]/_}"     # unique per invocation -> NO .out collision
  if ! command -v "$tool" >/dev/null 2>&1; then
    printf '%s\tabsent\t\t0\n' "$label" >> "$DET/_table.tsv"   # loud-by-counting; never a pass
    return 0
  fi
  # Subshell cd: deterministic CWD ($PROJECT_DIR) for relative-path tools WITHOUT
  # leaking the chdir into the rest of the pipeline. timeout bounds every tool.
  ( cd "$PROJECT_DIR" && timeout "$to" "$@" ) > "$DET/$slug.out" 2>&1
  local rc=$?                               # captured BEFORE any other command clobbers $?
  if [ "$rc" -eq 124 ]; then               # timeout's SIGTERM deadline -> loud, not silent
    printf 'TIMEOUT after %ss\n' "$to" >> "$DET/$slug.out"
  fi
  printf '%s\tran\t%s\t%s\n' "$label" "$rc" "$(wc -l < "$DET/$slug.out")" >> "$DET/_table.tsv"
  return 0                                  # NEVER abort the pipeline on a tool's nonzero exit
}

run()      { _run_with "$RUN_TIMEOUT"  "$@"; }   # default-bounded tool
run_long() { _run_with "$TEST_TIMEOUT" "$@"; }   # build/test suites (wider cap)

na() {                                      # na <tool> <reason>  -> language/artifact not in diff
  printf '%s\tna\t\t0\n' "$1" >> "$DET/_table.tsv"
  printf '%s: N/A — %s\n' "$1" "$2" >> "$DET/_na.log"
}

# Fire a tool ONLY when its language/artifact is present in the diff.
changed_has() { grep -Eq "$1" "$DET/_files.txt"; }                      # predicate
changed_of()  { grep -E "$1" "$DET/_files.txt" | sed "s#^#$PROJECT_DIR/#"; }  # absolute paths

# Diff-scope file list (zsh + bash safe). UNQUOTED $(changed_of …) is a footgun:
# zsh does NOT word-split command substitutions by default, so the whole multi-line
# blob would arrive as ONE argv. Read line-by-line into the CHANGED array instead and
# always pass it quoted-splatted ("${CHANGED[@]}"). Empty array -> zero args (gated by
# changed_has upstream, so this stays non-empty in practice).
collect() {                                 # collect <regex> -> fills global CHANGED[]
  CHANGED=()
  local f
  while IFS= read -r f; do
    [ -n "$f" ] && CHANGED+=("$f")
  done < <(changed_of "$1")
}
```

Invariants:
- One row per attempted invocation (full command in col1). Absent tools still get a row
  (`absent`), so the denominator the verifier checks is complete.
- Each invocation maps 1:1 to its own `.out` via the slug; `go vet`, `go build`, `go test`
  (and the three `cargo` subcommands) never overwrite each other's captured output.
- Output capped at `.out`; the table stores only `exit` + line-count. The model reads the
  `.out` to extract findings (§5), it does **not** invent counts.
- Every tool runs **from `$PROJECT_DIR`** (subshell `cd`, never leaked) and **under
  `timeout`** (`RUN_TIMEOUT`, or `TEST_TIMEOUT` via `run_long`). A timeout = exit 124 =
  derived `failed` with note `timeout` — counted, never silently dropped.
- Multi-file invocations expand via `collect` + `"${CHANGED[@]}"` (zsh-safe array), never
  unquoted `$(changed_of …)` — zsh would pass the whole blob as a single argv.

## 2. Command matrix (fire only on extensions/artifacts present in diff)

Each block is gated by `changed_has`. Inside the gate, `run()` self-handles absence (loud).
Languages **absent from the diff** are recorded once via `na` for an honest manifest.

```bash
# ---- Static analysis / idioms (per language) ----------------------------------------
# Diff-scope where the tool accepts paths; whole-package only where it must
# (go/cargo are package-graph tools — './...' is correct, not laziness).
if changed_has '\.go$'; then
  run golangci-lint run ./...; run go vet ./...; run staticcheck ./...
fi
if changed_has '\.rs$'; then
  run cargo clippy --all-targets -- -D warnings
fi
if changed_has '\.py$'; then
  collect '\.py$'
  run ruff check "${CHANGED[@]}"; run mypy "${CHANGED[@]}"
fi
if changed_has '\.(ts|tsx|js|jsx)$'; then
  collect '\.(ts|tsx|js|jsx)$'
  run eslint "${CHANGED[@]}"; run tsc --noEmit          # tsc needs the project graph
fi
if changed_has '\.(c|h)$|\.(cpp|hpp|cc|cxx)$'; then
  collect '\.(c|h|cpp|hpp|cc|cxx)$'
  run clang-tidy "${CHANGED[@]}"
  run cppcheck --enable=all "${CHANGED[@]}"
fi
if changed_has '\.sh$';        then collect '\.sh$'; run shellcheck "${CHANGED[@]}"; fi
if changed_has '\.rb$';        then run rubocop; fi
if changed_has '\.php$';       then run phpstan analyse; fi
if changed_has '\.kt$';        then run detekt; fi
if changed_has '\.swift$';     then run swiftlint; fi

# ---- IaC / CI / containers ----------------------------------------------------------
if changed_has '\.tf$';                         then run tflint; run tfsec .; run checkov -d .; fi
if changed_has '\.ya?ml$';                      then run yamllint .; fi
if changed_has '(^|/)\.github/workflows/.*\.ya?ml$'; then run actionlint; fi
if changed_has '(^|/)Dockerfile';               then collect 'Dockerfile'; run hadolint "${CHANGED[@]}"; fi
if changed_has '\.ya?ml$'; then
  collect '\.ya?ml$'
  if grep -rqsl 'apiVersion:' "${CHANGED[@]}" 2>/dev/null; then
    run kube-linter lint .; run kubeconform "${CHANGED[@]}"
  fi
fi
if changed_has '(playbook|roles/|tasks/).*\.ya?ml$'; then run ansible-lint; fi

# ---- Data / schema ------------------------------------------------------------------
if changed_has '\.sql$';   then collect '\.sql$'; run sqlfluff lint "${CHANGED[@]}"; fi
if changed_has '\.proto$'; then run buf lint; fi

# ---- Security (taint/SAST + secrets) ------------------------------------------------
# Secret/SAST scans stay repo-wide on purpose: a secret or taint sink can live
# OUTSIDE the diff yet be reachable from it. Diff-scoping these would create blind spots.
run semgrep --config auto --error                 # broad — fire on any code diff
run gitleaks detect --no-banner
run trufflehog filesystem .
run detect-secrets scan

# ---- Supply chain (lockfiles + manifests) -------------------------------------------
if changed_has '(package-lock\.json|yarn\.lock|pnpm-lock\.yaml|go\.(mod|sum)|Cargo\.lock|poetry\.lock|requirements.*\.txt)$'; then
  run osv-scanner -r .; run trivy fs .
fi
if changed_has '\.go$|go\.(mod|sum)$'; then run govulncheck ./...; fi

# ---- BUILD + EXISTING TESTS (catch breakage even with no CI) ------------------------
# run_long => wider TEST_TIMEOUT; a timeout still lands as exit 124 -> failed (note 'timeout').
if [ -f "$PROJECT_DIR/Makefile" ] && grep -qE '^test:' "$PROJECT_DIR/Makefile"; then
  run_long make test                              # make is present here
fi
if changed_has '\.go$';   then run go build ./...; run_long go test ./...; fi   # present
if changed_has '\.rs$';   then run cargo build;    run_long cargo test;    fi   # present
if changed_has '\.py$';   then run_long pytest -q; fi                           # absent here -> loud
```

**N/A bookkeeping** (optional but recommended for a clean manifest): for the dominant
scanner of each language NOT seen in the diff, emit `na <tool> "no <lang> files in diff"`
so `tiers[]` distinguishes "unsupported language" from "tool absent".

## 3. Render the table (awk — the model authors nothing here)

```bash
render_tier_table() {
  awk -F'\t' '
    BEGIN { printf "%-28s %-8s %-5s %-7s %s\n","INVOCATION","STATUS","EXIT","LINES","NOTE" }
    {
      tool=$1; st=$2; ex=$3; ln=$4; n=""
      if      (st=="absent")            { d="ABSENT"; n="!! not installed — dimension UNVERIFIED, NOT a pass"; ab++ }
      else if (st=="na")                { d="n/a";    n="not applicable to diff languages"; na++ }
      else if (st=="ran" && ex+0!=0)    { d="FAILED"; n="exit "ex" — findings ingested as HARD BLOCKERS"; fl++ }
      else                              { d="ran";    n="clean (exit 0)"; ok++ }
      printf "%-28s %-8s %-5s %-7s %s\n", tool, d, (ex==""?"-":ex), (ln==""?"-":ln), n
    }
    END { printf "\n%d clean · %d FAILED (blocking) · %d ABSENT (confidence-capped, named above) · %d n/a\n", ok, fl, ab, na }
  ' "$DET/_table.tsv"
}
render_tier_table        # paste verbatim into the report; the verifier re-parses _table.tsv
```

The verifier (`review-verify-manifest.sh`) independently re-parses `_table.tsv` and the
`.out` files; if the report's numbers diverge from the captured ones, the run is INVALID.

## 4. tier_status_model (the four states — verbatim contract)

```yaml
tier_status_model:                 # MUST appear in the report, generated from _table.tsv
  ran:    "tool executed; exit + finding count are REAL (parsed from .out)."
  absent: "tool not installed -> dimension marked 'unverified (tool absent)'. NOT a failure,
           NOT a pass. Surfaced LOUDLY (counted in the footer). Does not by itself block convergence."
  na:     "tool not applicable to this diff's languages -> 'N/A (unsupported language)'."
  failed: "exit != 0 with findings -> findings ingested as FIRST-CLASS hard-blocking candidates."
  rules:
    - "Tool findings are deduped vs LLM findings; a tool-confirmed finding gets a confidence boost."
    - "CI / make test / make lint logs ingested as dual-validation hard blockers."
    - "A genuinely-applicable tool that is 'absent' lowers MAX achievable confidence for that
       dimension and is named in the report — but the run can still reach a valid (qualified) verdict."
    - "absent != na != pass. Treating an absent tool as a pass is a FORBIDDEN guardrail violation."
    - "'failed' is derived (ran + exit!=0), never written as a raw status; raw statuses are ran|absent|na."
```

State decision (mechanical):

| Condition | Status | Effect on verdict |
|-----------|--------|-------------------|
| `command -v` succeeds, exit 0 | `ran` (clean) | dimension verified-clean for that tool |
| `command -v` succeeds, exit !=0 | `failed` | findings -> hard-blocking candidates (Phase 7) |
| `command -v` fails, language in diff | `absent` | dimension **unverified**, caps confidence, named loudly |
| language/artifact not in diff | `na` | dimension N/A (unsupported language) |

## 5. Tool findings vs LLM findings — dedup + confidence boost

Tool `.out` files are parsed into the **same finding schema** the executors emit (see
`triage.md`), tagged with their source, then merged in Phase 7's `generate-then-filter`
pipeline. Tool findings never replace LLM reasoning; they **corroborate or contradict** it.

```yaml
det_finding_ingestion:
  parse:
    - "Read each $DET/<slug>.out; extract {file, line, rule_id, message} per tool's format."
    - "Map to finding: {source:'det:<tool>', category, file, line, title, evidence:<rule_id + line>,
        confidence_pct: 90}  # deterministic tools start HIGH-confidence (grounded by definition)."
  dedup_vs_llm:
    key: "{category}:{file}:{line}:{normalized_title}"      # same key as Phase 7 dedup
    on_match: |
      An LLM finding and a det finding collide on the key ->
        keep the LLM finding (richer narrative + counterexample),
        add  confirmed_by: ["det:<tool>"],
        boost confidence +10 per confirming source (cap 100),
        label "multi-tier-confirmed".  (Mirror triage.md cross_tier_dedupe.)
    det_only: |
      A det finding with no LLM match -> surfaced on its own at its tool confidence (>=90);
      it is NOT dropped just because no agent independently saw it. Deterministic ground truth wins.
    llm_only_contradicted: |
      An LLM finding whose location a relevant tool scanned and reported CLEAN ->
        demote to Needs-Verification with note "det:<tool> did not corroborate" (a clean tool
        run is evidence; but 'tool absent' is NOT — absence corroborates nothing).
  failed_tier:
    rule: "Any tool with derived status=failed -> its parsed findings are HARD-BLOCKING
           candidates (dual-validation). make test / go test / cargo test failures are
           ingested the same way and gate the verdict."
```

A det-confirmed finding is the strongest evidence class in the whole pipeline: it carries
a real rule id, a real line, and a real nonzero exit. The Judge (Phase 7.5) cannot demote
it for "weak confidence".

## Guardrails (quick reference)

| Action | Status |
|--------|--------|
| Author tier exit codes / finding counts by hand | FORBIDDEN — parse `_table.tsv` + `.out` |
| Treat an `absent` tool as a pass | FORBIDDEN — `absent != na != pass`, confidence-capped, named loudly |
| Abort the pipeline on a tool's nonzero exit | FORBIDDEN — `run()` returns 0; nonzero -> `failed` finding |
| Silently skip a tool that is missing | FORBIDDEN — record `absent` row so the verifier sees it |
| Drop a deterministic finding for "low confidence" | FORBIDDEN — det findings start >=90, ground-truth class |
| Substitute `sg` for `ast-grep` | FORBIDDEN — `/usr/bin/sg` is `newgrp` |
| Fire a language tool when that language is not in the diff | Avoid — record `na` instead |
| Report build/test "skipped" when the tool is absent | FORBIDDEN — report `absent` (e.g. pytest here) |
| Run a tool without `timeout` | FORBIDDEN — every tool goes through `run`/`run_long` (bounded; 124 -> failed) |
| Run a tool from the wrong CWD | FORBIDDEN — `run()` subshell-`cd`s to `$PROJECT_DIR`; never `cd` the parent shell |
| Pass `$(changed_of …)` unquoted | FORBIDDEN — zsh won't word-split it; use `collect` + `"${CHANGED[@]}"` |
| Leave a literal `<changed>` / `<Dockerfile>` placeholder in a command | FORBIDDEN — use real `collect`/`"${CHANGED[@]}"` substitution |
