# Troubleshooting

## Common Issues

### The container does not start

**Symptom**: error on `Reopen in Container`

1. Check that Docker is running: `docker ps`
2. Check disk space: `docker system df`
3. Clean up if needed: `docker system prune`
4. Rebuild without cache: `Ctrl+Shift+P` → `Dev Containers: Rebuild Without Cache`

### RTK is not rewriting commands

**Symptom**: `rtk gain` shows zero savings, or commands run raw

1. Check the binary: `rtk --version` (must be ≥ 0.23.0)
2. Check the hook: `grep rtk-rewrite.sh ~/.claude/settings.json`
3. Check `jq` is installed (the hook needs it): `command -v jq`
4. Check stderr for `[rtk] WARNING:` lines on session start
5. If still broken: `rm -rf ~/.cache/rtk && rtk init`

### MCP tokens are not recognized

**Symptom**: `mcp__github__*` returns auth errors

1. Check that `.devcontainer/.env` contains the correct token
2. Rebuild the container (`.env` is read at startup)
3. Check the generated `mcp.json`: `cat /workspace/mcp.json`

### Post-edit hooks fail

**Symptom**: lint/format errors after each edit

1. The hook is **non-blocking**: the error is displayed but does not block Claude
2. If a linter is not installed for your language, the hook detects this and skips it
3. To temporarily disable a hook: edit `~/.claude/settings.json`

### VPN does not connect

**Symptom**: `/vpn --connect` fails

1. Check that `OP_SERVICE_ACCOUNT_TOKEN` is configured
2. Check that the VPN profile exists in 1Password: `/vpn --list`
3. Check Docker capabilities: `NET_ADMIN` and `/dev/net/tun` must be enabled (this is the default in `docker-compose.yml`)

### Missing Claude files at startup

**Symptom**: commands or agents not found

`postStart.sh` restores files from `/etc/claude-defaults/`. If the issue persists:

```bash
# Manual restoration
cp -r /etc/claude-defaults/* ~/.claude/
```

### Commit is blocked

**Symptom**: `commit-validate.sh` rejects the commit

The hook blocks commits containing AI mentions (Co-Authored-By Claude, Generated by AI, etc.). This is intentional.

If `security.sh` blocks the commit, it means a secret was detected in the staged files. Check with `git diff --cached` and remove the secret.

## Quick Checks

```bash
# General status
docker ps                           # Container running?
cat /workspace/mcp.json             # MCP configured?
echo $SHELL                         # Should be /bin/zsh
ls ~/.claude/commands/              # Commands present?
ls ~/.claude/agents/                # Agents present?

# Services
curl -s http://host.docker.internal:11434/api/tags  # Ollama?
op --version                        # 1Password CLI?
```
